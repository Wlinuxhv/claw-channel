## 🔍 问题发现

### 实际问题

在使用 `ssh-win10-android-builder` skill 构建 Claw Channel Android APK 时遇到：

**症状**:
```bash
# SCP 下载 APK
scp user@192.168.192.151:'C:\Users\User\claw-channel\Android\app\build\outputs\apk\debug\app-debug.apk' ./

# 远程文件大小
ssh user@192.168.192.151 'powershell -Command "(Get-Item ''...app-debug.apk'').Length"'
# 输出：14375089 bytes (14.4 MB)

# 本地文件大小
ls -lh app-debug.apk
# 输出：200K（明显小于远程！）
```

**根本原因**: Windows OpenSSH SCP 服务器对大文件（>10MB）传输支持不完善，导致文件损坏。

**影响**: 
- APK 文件损坏，无法安装
- 需要重新下载
- 浪费时间和带宽

---

## 💡 解决方案

### 核心改进：SFTP 优先策略

| 文件大小 | 原方案 | 新方案 | 改进 |
|----------|--------|--------|------|
| <5 MB    | SCP    | SCP（可选 SFTP） | 保持兼容 |
| 5-10 MB  | SCP    | **SFTP（推荐）** | 新增建议 |
| >10 MB   | SCP    | **SFTP（必须）** | ❗关键修复 |
| APK 文件 | SCP    | **SFTP（必须）** | ❗关键修复 |

   - paramiko 未安装解决
   - SFTP 连接被拒绝解决

3. **文件大小验证脚本**
