## 一、项目背景

用户需要一个类似阿里钉钉的即时通讯聊天软件（Android APK），通过**自建的 Go 服务端**与 **OpenClaw AI 个人助理**进行对话和交互。

**OpenClaw** 是一个开源的个人 AI 助手系统（https://github.com/openclaw/openclaw），代号"龙虾"（Molty - 太空龙虾 AI 助手 🦞）。

---

## 二、核心概念

| 概念 | 说明 |
|------|------|
| OpenClaw Gateway | AI 助手控制平面，提供 AI 对话能力 |
| Go 服务端 | 用户自建的服务端，负责用户认证、消息转发 |
| Claw Channel | 本项目的 Android 聊天客户端 |
| AppKey | 应用标识，由 Go 服务端生成并分发给用户 |
| Client Secret | 应用密钥，由 Go 服务端生成并分发给用户 |

**5 人小团队使用**

## 五、核心功能需求

### 5.1 激活登录
- 无需注册账号
- 首次使用需输入 **AppKey + Client Secret** 激活（由管理员分发）
- 激活后获取 Token，本地保存登录状态
- Token 过期自动刷新

### 5.2 聊天功能（类钉钉）
- 支持文字消息收发
- 支持图片消息收发
- 消息实时推送，低延迟（WebSocket）
- 聊天记录本地保存，可查看历史消息
- 显示消息发送状态（发送中/已发送/已读）
- 支持消息引用/回复

### 5.3 AI 对话功能
- 与 OpenClaw AI 进行自然语言对话
- 支持多轮对话上下文
- 支持发送控制命令（如：/status、/new、/reset 等）
- 显示 AI 思考和响应状态
- 支持显示 AI 回复中的富文本（代码块、链接等）

### 5.4 消息通知
- 接收 AI 主动推送的通知
- 支持后台消息推送（应用关闭时也能收到通知）
- 重要通知支持声音/震动提醒

### 5.5 设置功能
- AppKey/Secret 管理（查看/更换）
- Token 状态查看
- 通知开关设置
- 聊天记录清理
- 关于页面（版本信息等）
- 服务端地址配置（支持自定义）

## 六、认证流程

### 6.1 AppKey/Secret 生成（管理员）

管理员在 Go 服务端生成 5 套 AppKey + Secret，分发给 5 个用户：

```
用户 1: AppKey=claw_user_001, Secret=xxx
用户 2: AppKey=claw_user_002, Secret=xxx
用户 3: AppKey=claw_user_003, Secret=xxx
用户 4: AppKey=claw_user_004, Secret=xxx
用户 5: AppKey=claw_user_005, Secret=xxx
```

### 6.2 客户端激活（用户）

```
1. 用户打开 Claw Channel
2. 输入 AppKey + Client Secret
3. 客户端调用 Go 服务端认证接口
4. 服务端验证通过，返回 Token
5. 客户端保存 Token，进入聊天界面
```

### 6.3 认证接口（Go 服务端）

**请求**：
```http
POST /api/auth/login
Content-Type: application/json

{
  "app_key": "claw_user_001",
  "client_secret": "xxx"
}
```

**响应**：
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "access_token": "eyJhbGciOiJIUzI1NiIs...",
    "token_type": "Bearer",
    "expires_in": 7200,
    "refresh_token": "dGhpcyBpcyBhIHJlZnJl..."
  }
}
```

### 6.4 WebSocket 连接

```javascript
// 连接 URL
wss://claw.yourdomain.com/ws/chat?token=ACCESS_TOKEN

// 或使用 Header
wss://claw.yourdomain.com/ws/chat
Headers: Authorization: Bearer ACCESS_TOKEN
```

## 八、Go 服务端职责

| 功能 | 说明 |
|------|------|
| **用户认证** | 验证 AppKey + Secret，生成 Token |
| **Token 管理** | Token 生成、验证、刷新、过期处理 |
| **WebSocket** | 与客户端保持长连接，推送消息 |
| **消息转发** | 客户端消息 → OpenClaw → AI 回复 → 客户端 |
| **会话管理** | 管理每个用户的会话上下文 |
| **连接管理** | 维护与 OpenClaw Gateway 的连接 |

## 九、非功能需求

### 9.1 性能要求
| 指标 | 要求 | 说明 |
|------|------|------|
| 消息发送延迟 | < 1 秒 | 聊天消息 |
| AI 响应延迟 | 取决于模型 | 通常 2-10 秒 |
| 应用启动时间 | < 3 秒 | 冷启动 |
| 本地消息存储 | ≥ 1000 条 | SQLite 存储 |
| 服务端并发 | ≥ 5 人 | 小团队使用 |

### 9.2 安全要求
- AppKey + Secret 加密存储
- 通信数据加密（WSS/HTTPS）
- 本地数据加密存储
- Token 自动刷新机制
- 服务端限流（防止滥用）

### 9.3 可用性要求
- 支持后台运行，消息推送不中断
- 网络断开后自动重连
- 支持弱网环境使用
- Token 过期自动刷新

### 9.4 兼容性要求
- Android 8.0 及以上版本
- 支持主流安卓手机品牌（华为、小米、OPPO、vivo 等）

## 十、部署架构

```
┌──────────────────┐                          ┌──────────────────┐                          ┌──────────────────┐
│   安卓客户端      │                          │   您的 Go 服务端    │                          │   OpenClaw       │
│  (Claw Channel)  │                          │  (家庭服务器)      │                          │   Gateway        │
│  (用户手机)      │ ◄──── 公网域名 ─────────►│                    │ ◄──── 局域网 ──────────►│  (本地/远程)     │
│                  │      Cloudflare 隧道       │  - Docker 部署     │      或本地调用         │                  │
└──────────────────┘                          └──────────────────┘                          └──────────────────┘
```

**部署说明**：
- Go 服务端部署在用户家中（Docker 或直接运行）
- 通过 Cloudflare Tunnel 暴露公网访问
- 使用自定义域名访问
- OpenClaw Gateway 可能在同一台服务器或远程
