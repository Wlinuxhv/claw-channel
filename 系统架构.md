## 一、系统概述

### 1.1 系统定位

Claw Channel 是一个 Android 聊天客户端应用，作为 **OpenClaw AI 个人助理** 的一个 Channel（渠道），通过自建的 Go 服务端与 OpenClaw Gateway 进行通信。

### 1.2 系统架构

```
┌──────────────────┐          ┌──────────────────┐          ┌──────────────────┐
│   安卓客户端      │          │   Go 服务端       │          │   OpenClaw       │
│  (Claw Channel)  │          │  (家庭服务器)     │          │   Gateway        │
│  (5 个用户手机)   │          │                    │          │  (AI 服务)        │
│                  │          │  - 用户认证        │          │                  │
│  - AppKey/Secret │◄────────►│  - 消息转发        │◄────────►│  - AI 对话        │
│  - 聊天界面      │  WebSocket│  - 会话管理        │  WebSocket│  - Channel 管理   │
│  - 本地存储      │  :8080   │  - Token 管理      │  :18789  │                  │
└──────────────────┘          └──────────────────┘          └──────────────────┘
         │                            │
         │                            │
         └────────────────────────────┘
              Cloudflare Tunnel
            (公网访问 Go 服务端)
```

### 1.3 技术栈

| 组件 | 技术选型 | 说明 |
|------|----------|------|
| **Android 客户端** | Kotlin + Jetpack Compose | 现代 Android UI |
| **Go 服务端** | Go 1.21+ | 高性能后端 |
| **WebSocket** | gorilla/websocket (Go) + OkHttp (Android) | 实时通信 |
| **数据库** | SQLite (客户端) + SQLite/PostgreSQL (服务端) | 本地/服务端存储 |
| **部署** | Docker | 容器化部署 |

## 三、接口设计

### 3.1 认证接口（Go 服务端）

#### 3.1.1 登录激活

**请求**：
```http
POST /api/auth/login
Content-Type: application/json

{
  "app_key": "claw_user_001",
  "client_secret": "xxx"
}
```

**响应**：
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "access_token": "eyJhbGciOiJIUzI1NiIs...",
    "token_type": "Bearer",
    "expires_in": 7200,
    "refresh_token": "dGhpcyBpcyBhIHJlZnJl..."
  }
}
```

#### 3.1.2 Token 刷新

**请求**：
```http
POST /api/auth/refresh
Content-Type: application/json

{
  "refresh_token": "dGhpcyBpcyBhIHJlZnJl..."
}
```

**响应**：
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "access_token": "new_eyJhbGciOiJIUzI1NiIs...",
    "expires_in": 7200
  }
}
```

---

### 3.2 WebSocket 接口（Go 服务端 ↔ Android 客户端）

#### 3.2.1 连接 URL

```
wss://claw.yourdomain.com/ws/chat?token=ACCESS_TOKEN
```

或使用 Header：
```
wss://claw.yourdomain.com/ws/chat
Authorization: Bearer ACCESS_TOKEN
```

#### 3.2.2 客户端 → 服务端消息格式

```json
{
  "type": "message",
  "content": "你好，龙虾！",
  "content_type": "text",
  "session_id": "uuid",
  "message_id": "uuid",
  "timestamp": 1710768000
}
```

#### 3.2.3 服务端 → 客户端消息格式

```json
{
  "type": "message",
  "content": "你好！有什么我可以帮你的吗？",
  "content_type": "text",
  "session_id": "uuid",
  "message_id": "uuid",
  "reply_to": "uuid",
  "timestamp": 1710768000,
  "status": "complete"
}
```

#### 3.2.4 消息类型

| type | 说明 | 方向 |
|------|------|------|
| `message` | 普通聊天消息 | 双向 |
| `command` | 控制命令（/status 等） | 客户端→服务端 |
| `status` | 状态通知 | 服务端→客户端 |
| `error` | 错误消息 | 服务端→客户端 |
| `heartbeat` | 心跳包 | 双向 |

### 3.3 WebSocket 接口（Go 服务端 ↔ OpenClaw Gateway）

#### 3.3.1 连接 URL

ws://localhost:18789/ws

    id INTEGER PRIMARY KEY AUTOINCREMENT,

    content TEXT NOT NULL,
    content_type TEXT DEFAULT 'text',
    timestamp INTEGER NOT NULL,
    status TEXT DEFAULT 'sent',
    reply_to TEXT,

CREATE INDEX idx_messages_session ON messages(session_id);

```sql
CREATE TABLE config (
    key TEXT PRIMARY KEY,

```

---

### 4.2 Go 服务端数据库

#### 4.2.1 用户表 (users)

```sql
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    app_key VARCHAR(64) NOT NULL UNIQUE,
    client_secret_hash VARCHAR(255) NOT NULL,
    is_active BOOLEAN DEFAULT true,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### 4.2.2 Token 表 (tokens)

```sql
CREATE TABLE tokens (
    id SERIAL PRIMARY KEY,
    user_id INTEGER REFERENCES users(id),
    access_token_hash VARCHAR(255) NOT NULL UNIQUE,
    refresh_token_hash VARCHAR(255) NOT NULL UNIQUE,
    expires_at TIMESTAMP NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_tokens_access ON tokens(access_token_hash);
CREATE INDEX idx_tokens_refresh ON tokens(refresh_token_hash);
```

#### 4.2.3 消息日志表 (message_logs)

```sql
CREATE TABLE message_logs (
    id SERIAL PRIMARY KEY,
    user_id INTEGER REFERENCES users(id),
    session_id VARCHAR(64) NOT NULL,
    message_id VARCHAR(64) NOT NULL,
    content TEXT NOT NULL,
    direction VARCHAR(10) NOT NULL, -- 'inbound' or 'outbound'
    status VARCHAR(20) DEFAULT 'sent',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_message_logs_user ON message_logs(user_id);
CREATE INDEX idx_message_logs_session ON message_logs(session_id);
```

## 六、安全设计

### 6.1 认证安全

- **AppKey + Secret**：服务端生成，分发给用户
- **Token 机制**：JWT Token，有效期 2 小时
- **Refresh Token**：用于无感刷新，有效期 7 天
- **密码加密**：bcrypt 加密存储（成本因子 12）

### 6.2 通信安全

- **WSS/HTTPS**：所有通信加密
- **Token 验证**：每次请求验证 Token
- **防重放攻击**：消息带时间戳，服务端验证

### 6.3 数据安全

- **本地加密**：Android 本地数据库加密（SQLCipher）
- **敏感信息**：Token 加密存储
- **日志脱敏**：敏感信息不记录日志

## 七、性能设计

### 7.1 性能指标

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 消息延迟 | < 500ms | 客户端→服务端→客户端 |
| AI 响应延迟 | 2-10s | 取决于 OpenClaw |
| 并发连接 | ≥ 10 | 支持 5 用户 + 冗余 |
| 消息存储 | ≥ 1000 条/用户 | 本地 SQLite |

### 7.2 优化措施

- **连接池**：Go 服务端维护 OpenClaw 连接池
- **消息队列**：异步处理消息转发
- **本地缓存**：Android 本地缓存最近消息
- **心跳机制**：30 秒心跳，保持连接活跃

## 八、部署方案

### 8.1 Go 服务端部署

#### Docker Compose 配置

```yaml
version: '3.8'

services:
  claw-server:
    build: ./server
    container_name: claw-server
    restart: unless-stopped
    ports:
      - "8080:8080"
    environment:
      - DATABASE_URL=sqlite:///data/claw.db
      - OPENCLAW_WS_URL=ws://openclaw:18789/ws
      - JWT_SECRET=your-secret-key
    volumes:
      - ./data:/data
    networks:
      - claw-network

  openclaw:
    image: openclaw/openclaw:latest
    container_name: openclaw
    restart: unless-stopped
    ports:
      - "18789:18789"
    environment:
      - OPENCLAW_CONFIG=/config/openclaw.json
    volumes:
      - ./openclaw-config:/config
      - ./openclaw-data:/data
    networks:
      - claw-network

networks:
  claw-network:
    driver: bridge
```

### 8.2 Cloudflare Tunnel 配置

```yaml
# config.yml
tunnel: your-tunnel-id
credentials-file: /root/.cloudflared/credentials.json

ingress:
  - hostname: claw.yourdomain.com
    service: http://claw-server:8080
  - service: http_status:404
```

### 8.3 Android 客户端发布

- **APK 打包**：Release 签名
- **分发方式**：直接分发 APK 文件
- **更新机制**：应用内检查更新
