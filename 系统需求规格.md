## 一、引言

### 1.1 项目概述

**Claw Channel** 是一个 Android 聊天客户端应用，用于与 **OpenClaw AI 个人助理**进行对话和交互。系统采用分布式架构，支持多 OpenClaw 实例接入，通过 ZeroTier 虚拟局域网实现安全通信。

### 1.2 项目目标

- 为 5 人小团队提供类钉钉的聊天体验
- 支持多机器人管理（每个机器人对应一个 OpenClaw 实例）
- 通过推荐码机制实现用户激活
- 支持 Android 和鸿蒙系统

### 1.3 适用范围

本文档描述系统的功能需求、非功能需求、接口需求和数据需求，是系统设计、开发和测试的依据。

### 1.4 术语定义

| 术语 | 定义 |
|------|------|
| OpenClaw | 开源个人 AI 助手系统（https://github.com/Wlinuxhv/openclaw） |
| Go 服务端 | 用户自建的服务端，负责用户认证、消息转发 |
| Claw Channel | Android 聊天客户端应用 |
| 推荐码 | 8 位 UUID 短码，用于用户激活 APP 账号 |
| 机器人 | 一个 AI 助理实例，由 AppKey+Secret+OpenClaw 地址配置 |
| ZeroTier | 虚拟局域网技术，用于连接分布式 OpenClaw 实例 |

## 三、功能需求

### 3.1 用户认证模块

#### 3.1.1 推荐码激活

**FR-001**: 用户首次打开 App 时，显示推荐码输入界面

**FR-002**: 推荐码格式为 8 位 UUID 短码（如 a1b2c3d4），大小写敏感

**FR-003**: 推荐码一次性使用，使用后失效

**FR-004**: 推荐码仅用于激活 APP 账号，不绑定特定机器人

**FR-005**: 激活成功后，自动登录并进入聊天界面

#### 3.1.2 登录状态管理

**FR-006**: 激活后登录状态永久有效，除非用户主动退出或清除数据

**FR-007**: Token 过期自动刷新（有效期 2 小时）

**FR-008**: 支持退出登录功能

**FR-009**: 支持切换账号（多账号管理）

---

### 3.2 聊天功能模块

#### 3.2.1 聊天界面

**FR-010**: 采用单一直聊 AI 界面布局（类似微信单人聊天）

**FR-011**: 界面顶部显示当前机器人名称

**FR-012**: 支持文字消息收发

**FR-013**: 支持图片消息收发（最大 100MB，自动压缩）

**FR-014**: 支持语音消息收发（按住说话）

**FR-015**: 支持文件传输（所有文件类型）

**FR-016**: 不支持链接预览

#### 3.2.2 消息状态

**FR-017**: 显示消息发送中状态（转圈动画）

**FR-018**: 显示消息成功发送状态（✓标记）

**FR-019**: 显示 AI 已读状态（特殊标记）

**FR-020**: 显示消息发送失败状态（红色感叹号）

#### 3.2.3 消息操作

**FR-021**: 长按消息弹出操作菜单

**FR-022**: 支持复制消息到剪贴板

**FR-023**: 支持引用/回复消息

**FR-024**: 支持重发失败的消息

**FR-025**: 不支持转发、收藏、置顶功能

#### 3.2.4 机器人切换

**FR-026**: 左上角菜单按钮，点击展开侧边栏

**FR-027**: 侧边栏显示所有可用机器人列表

**FR-028**: 点击机器人切换聊天对象

**FR-029**: 切换机器人后，聊天记录独立显示

## 四、非功能需求

### 4.1 性能需求

| 指标 | 要求 | 说明 |
|------|------|------|
| **消息发送延迟** | < 1 秒 | 聊天消息从发送到显示 |
| **AI 响应延迟** | 取决于模型 | 通常 2-10 秒 |
| **应用启动时间** | < 3 秒 | 冷启动时间 |
| **本地消息存储** | ≥ 1000 条/机器人 | SQLite 存储 |
| **服务端并发** | ≥ 10 连接 | 支持 5 用户 + 冗余 |
| **服务器配置** | 2C2G | 最低配置要求 |

### 4.2 安全需求

**NFR-001**: AppKey + Secret 加密存储（bcrypt）

**NFR-002**: 通信数据加密（WSS/HTTPS）

**NFR-003**: 本地数据加密存储（SQLCipher）

**NFR-004**: Token 自动刷新机制

**NFR-005**: 推荐码一次性使用

**NFR-006**: 管理员密码加密存储

**NFR-007**: ZeroTier 虚拟局域网隔离

### 4.3 可用性需求

**NFR-008**: 支持后台运行，消息推送不中断

**NFR-009**: 网络断开后自动重连

**NFR-010**: 支持弱网环境使用

**NFR-011**: Token 过期自动刷新

**NFR-012**: 服务端优雅关闭

### 4.4 兼容性需求

**NFR-013**: Android 8.0 及以上版本

**NFR-014**: 鸿蒙系统（标准 Android APK）

**NFR-015**: 主流安卓手机品牌（华为、小米、OPPO、vivo 等）

### 4.5 可维护性需求

**NFR-016**: 日志分级，方便调试

**NFR-017**: 配置外部化，支持环境变量

**NFR-018**: Docker 容器化部署

**NFR-019**: 监控指标可视化

## 五、接口需求

### 5.1 客户端 - 服务端接口

#### 5.1.1 认证接口

**POST /api/auth/login**

请求：
```json
{
  "recommendation_code": "a1b2c3d4"
}
```

响应：
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "access_token": "eyJhbGciOiJIUzI1NiIs...",
    "token_type": "Bearer",
    "expires_in": 7200,
    "refresh_token": "dGhpcyBpcyBhIHJlZnJl..."
  }
}
```

**POST /api/auth/refresh**

请求：
```json
{
  "refresh_token": "dGhpcyBpcyBhIHJlZnJl..."
}
```

响应：
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "access_token": "new_eyJhbGciOiJIUzI1NiIs...",
    "expires_in": 7200
  }
}
```

#### 5.1.2 WebSocket 接口

**WS /ws/chat**

连接 URL：
```
wss://claw.yourdomain.com/ws/chat?token=ACCESS_TOKEN
```

客户端→服务端消息：
```json
{
  "type": "message",
  "content": "你好",
  "content_type": "text",
  "session_id": "uuid",
  "message_id": "uuid",
  "timestamp": 1710768000
}
```

服务端→客户端消息：
```json
{
  "type": "message",
  "content": "你好！有什么我可以帮你的吗？",
  "content_type": "text",
  "session_id": "uuid",
  "message_id": "uuid",
  "reply_to": "uuid",
  "timestamp": 1710768000,
  "status": "complete"

```

## 六、数据需求

### 6.1 客户端数据库（SQLite）

#### 6.1.1 用户表 (users)

```sql
CREATE TABLE users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    recommendation_code TEXT NOT NULL,
    access_token TEXT NOT NULL,
    refresh_token TEXT NOT NULL,
    token_expires_at INTEGER NOT NULL,
    is_active INTEGER DEFAULT 1,
    created_at INTEGER NOT NULL,
    updated_at INTEGER NOT NULL
);
```

#### 6.1.2 机器人表 (robots)

```sql
CREATE TABLE robots (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    app_key TEXT NOT NULL,
    secret_hash TEXT NOT NULL,
    openclaw_url TEXT NOT NULL,
    is_active INTEGER DEFAULT 1,
    created_at INTEGER NOT NULL,
    updated_at INTEGER NOT NULL
);
```

#### 6.1.3 消息表 (messages)

```sql
CREATE TABLE messages (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    robot_id INTEGER NOT NULL,
    session_id TEXT NOT NULL,
    message_id TEXT NOT NULL UNIQUE,
    from_user TEXT NOT NULL,
    content TEXT NOT NULL,
    content_type TEXT DEFAULT 'text',
    file_path TEXT,
    file_size INTEGER,
    timestamp INTEGER NOT NULL,
    status TEXT DEFAULT 'sent',
    reply_to TEXT,
    created_at INTEGER NOT NULL
);

CREATE INDEX idx_messages_robot ON messages(robot_id);
CREATE INDEX idx_messages_timestamp ON messages(timestamp);
```

#### 6.1.4 推荐码表 (recommendation_codes)

```sql
CREATE TABLE recommendation_codes (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    code TEXT NOT NULL UNIQUE,
    is_used INTEGER DEFAULT 0,
    used_by INTEGER,
    created_at INTEGER NOT NULL,
    used_at INTEGER
);
```

### 6.2 服务端数据库（SQLite）

#### 6.2.1 用户表 (users)

```sql
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    recommendation_code TEXT NOT NULL UNIQUE,
    is_active BOOLEAN DEFAULT true,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### 6.2.2 机器人表 (robots)

```sql
CREATE TABLE robots (
    id SERIAL PRIMARY KEY,
    name VARCHAR(64) NOT NULL,
    app_key VARCHAR(64) NOT NULL UNIQUE,
    secret_hash VARCHAR(255) NOT NULL,
    openclaw_url VARCHAR(255) NOT NULL,
    is_active BOOLEAN DEFAULT true,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### 6.2.3 Token 表 (tokens)

```sql
CREATE TABLE tokens (
    id SERIAL PRIMARY KEY,
    user_id INTEGER REFERENCES users(id),
    access_token_hash VARCHAR(255) NOT NULL UNIQUE,
    refresh_token_hash VARCHAR(255) NOT NULL UNIQUE,
    expires_at TIMESTAMP NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### 6.2.4 推荐码表 (recommendation_codes)

```sql
CREATE TABLE recommendation_codes (
    id SERIAL PRIMARY KEY,
    code VARCHAR(16) NOT NULL UNIQUE,
    is_used BOOLEAN DEFAULT false,
    used_by INTEGER REFERENCES users(id),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    used_at TIMESTAMP
);
```

#### 6.2.5 消息日志表 (message_logs)

```sql
CREATE TABLE message_logs (
    id SERIAL PRIMARY KEY,
    user_id INTEGER REFERENCES users(id),
    robot_id INTEGER REFERENCES robots(id),
    session_id VARCHAR(64) NOT NULL,
    message_id VARCHAR(64) NOT NULL,
    content TEXT NOT NULL,
    direction VARCHAR(10) NOT NULL,
    status VARCHAR(20) DEFAULT 'sent',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## 七、部署需求

### 7.1 Go 服务端部署

**Docker Compose 配置**：

```yaml
version: '3.8'

services:
  claw-server:
    build:
      context: ./Server
      dockerfile: Dockerfile
    container_name: claw-server
    restart: unless-stopped
    ports:
      - "8080:8080"
    environment:
      - SERVER_PORT=8080
      - JWT_SECRET=${JWT_SECRET}
      - DATABASE_URL=sqlite:///data/claw.db
      - ZEROTIER_NETWORK_ID=${ZEROTIER_NETWORK_ID}
    volumes:
      - ./data:/data
      - ./logs:/logs
    networks:
      - claw-network
      - zerotier-network

networks:
  claw-network:
    driver: bridge
  zerotier-network:
    external: true
```

### 7.2 Android 客户端部署

- **打包格式**：APK
- **签名**：Release 签名
- **分发方式**：直接分发 APK 文件
- **更新机制**：应用内检查更新

### 7.3 ZeroTier 配置

- 所有 OpenClaw 实例加入同一 ZeroTier 网络
- Go 服务端加入同一 ZeroTier 网络
- 通过 ZeroTier IP 互相访问
