- **当前版本**：v0.1.0 (开发中)

## 项目目标

创建一个 Android 聊天客户端应用，作为 **OpenClaw AI 个人助理** 的一个 Channel，通过自建的 Go 服务端与 OpenClaw Gateway 进行通信，供 5 人小团队使用。

## 技术架构

```
┌──────────────────┐          ┌──────────────────┐          ┌──────────────────┐

│                  │          │  - 用户认证        │          │                  │
│  - AppKey/Secret │◄────────►│  - 消息转发        │◄────────►│  - AI 对话        │
│  - 聊天界面      │  WebSocket│  - 会话管理        │  WebSocket│  - Channel 管理   │
│  - 本地存储      │  :8080   │  - Token 管理      │  :18789  │                  │
└──────────────────┘          └──────────────────┘          └──────────────────┘
         │                            │
         │                            │
         └────────────────────────────┘
              Cloudflare Tunnel
            (公网访问 Go 服务端)
```

## 技术栈

### 服务端
- **语言**：Go 1.21+
- **WebSocket**：gorilla/websocket
- **JWT**：golang-jwt/jwt/v5
- **数据库**：SQLite / PostgreSQL
- **部署**：Docker

### 客户端
- **语言**：Kotlin
- **UI**：Jetpack Compose
- **架构**：MVVM + Hilt
- **数据库**：Room (SQLite)
- **网络**：OkHttp + WebSocket

## 关键决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 项目定位 | AI 助手聊天客户端 | OpenClaw 实际是 AI 助手系统 |
| 认证方式 | AppKey + Secret | 用户完全控制，类似钉钉 |
| 架构模式 | 三层架构 | 服务端可管理用户、转发消息 |
| 协议选择 | 自研 WebSocket | 开发成本低，家庭部署足够 |
| 技术拆解 | 架构师自行分析 | 用户明确要求 |
