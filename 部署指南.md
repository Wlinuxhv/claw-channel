# 部署指南

## 快速部署

### 前提条件

- Go 1.19+ 已安装
- 域名已配置（可选，用于公网访问）
- Cloudflare Tunnel 已配置（可选，用于内网穿透）
- **注意**：当前环境为容器环境，不使用 Docker

---

## 方案一：直接运行二进制文件（推荐）

### 1. 准备环境

```bash
cd "/app/working/Claw Channel/Server"
```

### 2. 配置环境变量

创建 `.env` 文件：

```bash
# JWT 密钥（生产环境请修改为随机字符串）
JWT_SECRET=your-super-secret-jwt-key-change-me

# 服务端端口
SERVER_PORT=8080

# 管理员密码
ADMIN_PASSWORD=admin123
```

### 3. 编译服务端

```bash
# 下载依赖
go mod download

# 编译
go build -o claw-server cmd/server/main.go
```

### 4. 启动服务

```bash
# 方式 1：直接运行
./claw-server

# 方式 2：后台运行
nohup ./claw-server > server.log 2>&1 &

# 方式 3：使用 systemd（Linux 服务器）
sudo systemctl start claw-server
```

### 5. 查看日志

```bash
# 查看实时日志
tail -f server.log
```

### 6. 停止服务

```bash
# 查找进程
ps aux | grep claw-server

# 停止进程
kill <PID>
```

## 方案二：使用部署脚本

### 1. 创建启动脚本

```bash
#!/bin/bash
# start.sh

cd "/app/working/Claw Channel/Server"

# 检查是否已运行
if pgrep -f "claw-server" > /dev/null; then
    echo "Server is already running"
    exit 0
fi

# 启动服务
nohup ./claw-server > server.log 2>&1 &

echo "Server started"
echo "PID: $!"
```

### 2. 创建停止脚本

```bash
#!/bin/bash
# stop.sh

pkill -f "claw-server"
echo "Server stopped"
```

### 3. 使用脚本

```bash
chmod +x start.sh stop.sh
./start.sh
./stop.sh
```

## 公网访问配置

### Cloudflare Tunnel 配置

#### 1. 安装 cloudflared

```bash
# Linux
wget https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-amd64
chmod +x cloudflared-linux-amd64
sudo mv cloudflared-linux-amd64 /usr/local/bin/cloudflared
```

#### 2. 创建 Tunnel

```bash
cloudflared tunnel create claw-channel
```

#### 3. 配置 Tunnel

创建 `~/.cloudflared/config.yml`：

```yaml
tunnel: your-tunnel-id
credentials-file: /root/.cloudflared/credentials.json

ingress:
  - hostname: claw.yourdomain.com
    service: http://localhost:8080
  - service: http_status:404
```

#### 4. 运行 Tunnel

```bash
cloudflared tunnel run claw-channel
```

#### 5. 配置 DNS

在 Cloudflare Dashboard 中添加 CNAME 记录：

```
claw.yourdomain.com CNAME your-tunnel-id.cfargotunnel.com
```

## 用户配置

### 配置 5 个用户

编辑 `Server/internal/config/config.go`：

```go
cfg.AllowedUsers["claw_user_001"] = "random_secret_001"
cfg.AllowedUsers["claw_user_002"] = "random_secret_002"
cfg.AllowedUsers["claw_user_003"] = "random_secret_003"
cfg.AllowedUsers["claw_user_004"] = "random_secret_004"
cfg.AllowedUsers["claw_user_005"] = "random_secret_005"
```

**建议**：使用随机字符串生成工具生成安全的 Secret：

```bash
# 生成随机字符串
openssl rand -hex 16
```

### 分发凭证

将 AppKey 和 Client Secret 分发给 5 个用户：

| 用户 | AppKey | Client Secret |
|------|--------|---------------|
| 用户 1 | claw_user_001 | random_secret_001 |
| 用户 2 | claw_user_002 | random_secret_002 |
| ... | ... | ... |

## 故障排查

### 问题 1：服务无法启动

**症状**：运行 `./claw-server` 后立即退出

**解决**：
1. 检查日志：`cat server.log`
2. 检查端口占用：`netstat -tlnp | grep 8080`
3. 检查权限：`chmod +x claw-server`
4. 检查 Go 环境：`go version`

### 问题 2：WebSocket 连接失败

**症状**：Android 客户端无法建立 WebSocket 连接

**解决**：
1. 检查 Token 是否有效
2. 检查网络连接
3. 检查服务端日志
4. 检查防火墙设置

### 问题 3：端口被占用

**症状**：`address already in use`

**解决**：
```bash
# 查找占用端口的进程
lsof -i :8080
# 或
netstat -tlnp | grep 8080

# 停止进程
kill -9 <PID>
```

### 问题 4：数据库锁死

**症状**：`database is locked`

**解决**：
1. 停止服务
2. 删除数据库锁文件：`rm data/claw.db-shm data/claw.db-wal`
3. 重启服务

## 安全建议

1. **修改默认密钥**：生产环境必须修改 JWT_SECRET
2. **使用 HTTPS/WSS**：通过 Cloudflare Tunnel 自动启用 HTTPS
3. **定期备份**：定期备份数据库和配置文件
4. **监控日志**：定期检查日志，发现异常及时处理
5. **限制访问**：如有需要，可配置 IP 白名单
