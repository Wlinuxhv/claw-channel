## 📝 技术说明

### JWT Token 机制

- **Access Token**: 2 小时有效期，用于 API 认证
- **Refresh Token**: 7 天有效期，用于刷新 Access Token
- **算法**: HS256
- **存储**: SQLite 数据库（hash 存储）

### 推荐码机制

- **格式**: 8 位 UUID 短码（如 a1b2c3d4）
- **有效期**: 一次性使用
- **用途**: 用户激活 APP 账号
- **生成**: 管理员后台生成

### WebSocket 连接

- **协议**: WSS（生产环境）/ WS（开发环境）
- **认证**: Token 认证（URL 参数或 Authorization Header）
- **心跳**: 待实现
- **重连**: 待实现

## ⚠️ 注意事项

1. **生产环境必须修改**：
   - `JWT_SECRET` - 使用强随机字符串
   - `ADMIN_PASSWORD` - 使用强密码

2. **数据库备份**：
   - 定期备份 `/data/claw.db`

3. **日志管理**：
   - 日志文件保留 30 天
   - 注意磁盘空间

4. **安全建议**：
   - 启用 HTTPS/WSS
   - 配置防火墙
   - 使用 ZeroTier 隔离 OpenClaw 网络
